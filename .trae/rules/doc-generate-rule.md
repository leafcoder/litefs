# 文档生成控制规则
1. **禁止自动生成冗余文档**：开发阶段（coding/debugging），禁止自动生成 PRD、详细设计、会议纪要等非核心文档。
2. **只生成必要文档**：仅在以下场景自动生成文档：
   - API 接口文档（必须与代码同步）
   - 项目 README（最简版）
   - 数据库 Schema 文档
3. **文档精简原则**：所有自动生成的文档必须控制在 **2 页以内**，详细内容需经手动确认后再扩展。
4. **指定输出目录**：所有自动生成的文档，**必须统一输出到 `/docs/auto-generated/` 目录**，不得散落在项目根目录或代码目录中。
5. **版本关联**：生成文档时，必须在头部注明 `Generated by TRAE SOLO at [时间]`，并关联当前 Git Commit ID。
