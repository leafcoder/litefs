# 热重载功能实现文档

## 概述

LiteFS 的热重载功能允许在开发环境下，当项目相关文件发生变化时自动检测并提示用户。系统会智能监控项目启动后的相关文件（Python 源文件、配置文件等），并在检测到变化时输出日志提示用户手动重启服务器。

## 工作原理

### 1. 智能文件监控

系统会自动监控以下目录：

1. **启动脚本所在目录**：项目主文件所在的目录
2. **Python 路径中的相关目录**：`sys.path` 中的用户代码目录（排除系统目录和虚拟环境）

监控时会递归监控所有子目录。

### 2. 文件类型过滤

`FileEventHandler` 类会智能判断文件类型，决定是否触发重载：

- **需要重新加载的文件**：
  - Python 源文件（`.py`）
  - 配置文件（`.yaml`, `.yml`, `.json`, `.ini`, `.conf`）

- **自动排除的文件**：
  - 日志文件（`.log`）
  - Python 编译文件（`.pyc`, `.pyo`, `.so`）
  - 临时文件和备份文件（`~`, `.bak`, `.tmp`, `.swp`, `.orig`）
  - 文件名中包含波浪号 `~` 的文件（编辑器备份文件）
  - `__pycache__` 目录中的文件
  - `.git`, `.svn`, `.hg` 目录中的文件
  - `node_modules` 目录中的文件

### 3. 编辑器保存行为处理

系统特别处理了编辑器保存文件时的常见行为：

当编辑器保存文件时，通常会：
1. 将原文件重命名为备份文件（如 `app.py` → `app.py~`）
2. 创建新的源文件（`app.py`）

这个过程中会产生多个文件事件，但系统会：
- **自动过滤备份文件相关的移动事件**
- **使用 1 秒冷却时间防止频繁触发**
- **只响应真正需要重载的文件变化**

### 4. 监控目录日志输出

启动时会输出监控的目录信息：

```
[2026/04/02 14:00:00] INFO Monitoring project directory: /path/to/project
[2026/04/02 14:00:00] INFO Monitoring path: /path/to/custom/module
```

### 5. 文件变化处理

当检测到文件变化时：

1. 判断文件类型是否在监控范围内
2. 如果是需要重新加载的文件：
   - 记录日志信息
   - 设置 `_need_reload` 标志
   - 清空所有缓存
   - 关闭服务器
   - 输出提示让用户手动重启
3. 如果是不需要重新加载的文件：
   - 仅清空缓存

## 使用示例

### 基本用法

```python
from litefs import Litefs
from litefs.routing import get

app = Litefs(host='0.0.0.0', port=8080, debug=True)

@get('/')
def index_handler(request):
    return {'message': 'Hello World'}

app.register_routes(__name__)

if __name__ == '__main__':
    app.run(poll_interval=0.1, processes=1)
```

### 项目结构示例

```
my_project/
├── app.py              # 启动脚本
├── config.yaml         # 配置文件（变化会触发重载）
├── handlers/
│   ├── __init__.py
│   └── user.py         # 处理器模块（变化会触发重载）
├── models/
│   ├── __init__.py
│   └── user.py         # 模型模块（变化会触发重载）
├── utils/
│   └── helpers.py      # 工具模块（变化会触发重载）
├── logs/
│   └── app.log         # 日志文件（不会触发重载）
└── __pycache__/        # 缓存目录（不会触发重载）
```

### 文件变化时的日志输出

当修改 Python 文件时：

```
[2026/04/02 14:00:00] INFO File modified: /path/to/my_project/handlers/user.py, reloading application...
[2026/04/02 14:00:00] INFO ============================================================
[2026/04/02 14:00:00] INFO File changes detected!
[2026/04/02 14:00:00] INFO Please restart the server manually to apply changes.
[2026/04/02 14:00:00] INFO Changed file: /path/to/my_project/app.py
[2026/04/02 14:00:00] INFO ============================================================
```

## 注意事项

1. **手动重启**：当前实现需要用户手动重启服务器以应用更改
2. **开发环境专用**：热重载功能主要用于开发环境，生产环境应禁用
3. **单进程模式**：热重载在单进程模式下工作最佳
4. **全局状态**：重启会重置全局变量和模块级状态
5. **Session 数据**：使用内存 Session 时，重启会导致 Session 数据丢失
6. **监控范围**：系统会自动排除系统目录、虚拟环境和缓存目录，避免不必要的监控

## 性能优化

1. **智能排除**：系统自动排除不需要的目录，减少监控开销
2. **文件类型过滤**：只监控相关类型的文件，避免频繁触发重载
3. **日志文件排除**：避免日志写入触发重载循环
